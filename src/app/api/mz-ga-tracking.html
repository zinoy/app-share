因为移动端与PC端监测代码有所区别, 所以分别对移动端与PC端监测代码进行封装, 引用以下js文件时, 不要搞混了。

<!-- 别克 移动端监测代码JS --> <script type="text/javascript" src="//m.buick.com.cn/js/mz_ga_code.min.js"></script> <!-- 别克 PC端监测代码JS --> <script type="text/javascript" src="//www.buick.com.cn/js/mz_ga_code.min.js"></script>
<!-- 雪佛兰 移动端监测代码JS --> <script type="text/javascript" src="//m.chevrolet.com.cn/js/mz_ga_code.min.js"></script> <!-- 雪佛兰 PC端监测代码JS --> <script type="text/javascript" src="//www.chevrolet.com.cn/js/mz_ga_code.min.js"></script>
API 说明
  • hylink.trackPV(mzPage, gaPage)
  • hylink.trackEvent(mzBtn, category, action, label)
  • hylink.setGaPath(path)
  • hylink.trackLeads(name, mobile, leadsId)

hylink.trackPV(mzPage, gaPage)

触发一次PV tracking

参数:

  • mzPage: String mzPV名称 如: 威朗GS X 理想家H5-首页
  • gaPage: String gaPV名称 如: /home

返回: String

无
//触发一次PV hylink.trackPV("威朗GS X 理想家H5-首页", "/home");

hylink.trackEvent(mzBtn, category, action, label)

触发一次Event tracking

参数:

  • mzBtn: String mz按键名称 如: 威朗GS X 理想家H5-返回首页
  • category: String ga类别名称 如: 顶部导航, 侧导航
  • action: String ga操作名称 如: 点击, 播放
  • label: String ga标签(按键)名称 如: 返回首页, 开始游戏

返回: String

无
//触发一次点击事件 hylink.trackEvent("威朗GS X 理想家H5-返回首页", "顶部导航", "点击", "返回首页");

hylink.setGaPath(path)

设定ga PV tracking路径(PV值的前缀), 同一页面只需定义一次前缀。
注: 默认会获取当前URL所在路径名称, 除非需要tracking ga的PV路径不是当前URL所在位置, 才需要用此方法去调用。

参数:

  • path: String mzPV名称 如: 威朗GS X 理想家H5-首页

返回: String

无
//定义ga tracking pv值的前缀 hylink.setGaPath("/2014promotion/excellent");

hylink.trackLeads(name, mobile, leadsId)

当用户提交试驾数据成功后, 后台程序会返回leadsId, 这时需要把name, mobile, leadsId交到mz 记录一次tracking。
注: 此方法已整合到testdrive接口, 所以不再需要单独调用。

参数:

  • name: String 试驾者的名字, 如: Zino
  • mobile: String 试驾者的手机号码, 如: 19461041344
  • leadsId: String 由后台程序返回的id, 如: CH47330

返回: String

无
//定义ga tracking leads值的前缀 hylink.trackLeads("zino", "19461041344", "CH47330");
实例

Account或PM会提供类似下图的excel文件, 按此文件上的代码添加既可。

  1. 图片下方黄色代码部分不需要理会, 这部分代码已整合至mz_ga_code.min.js当中。
  2. 页面名称 是 mz 监测名称。
  3. 监测类型 有三种: pv, 点击, leads。
  4. 所在页面地址 这个可以忽略, 只做参考。
  5. GA 是 ga 监测名称。添加代码时, 请不要把pv:/{countdown}或pv:/{PATH}这部分算到 ga PV值中去。

在HTML页面上引用JS代码

根据不同的品牌(别克&雪佛兰)与PC站或移动站 引用不同的JS代码

<!-- 雪佛兰 移动端监测代码JS --> <script type="text/javascript" src="//m.chevrolet.com.cn/js/mz_ga_code.min.js"></script>

添加一条PV代码

//页面上触发一次PV hylink.trackPV("Mobile端-乐风RV倒计时页面-loading", "/loading");

添加一条Event代码

//在按键上触发一次Event hylink.trackEvent("Mobile端-乐风RV倒计时页面-第三页-答案", "第三页", "点击", "答案");

添加一条Leads代码

//在按键上触发一次PV与leads hylink.trackPV("Mobile端-乐风RV倒计时页面-试驾页面-预约试驾成功", "/testdrive-succeed"); hylink.trackLeads("zino", "19461041344", "CH47330");

如果使用testdrive.js来提交试驾数据, 此方法已整合进testdrive.js, 所以你可以完全忽略上边两行代码, 只需要定义hylink.setTestdriveSuccessPV()方法既可

//如果使用testdrive.js来提交数据, 请忽略上边两行代码, 使用以下代码来定义既可 hylink.setTestdriveSuccessPV("Mobile端-乐风RV倒计时页面-试驾页面-预约试驾成功");
